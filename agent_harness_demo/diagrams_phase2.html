① 第二阶段 AgentRunner 增强闭环

在第一阶段闭环上叠加：模型调用走 ResilientModelGateway（含 retry/fallback），工具调用走 ToolRuntime（含 policy/audit），全部挂到 trace tree。

② ResilientModelGateway 错误治理

主模型重试（指数退避）→ 不可重试错误立即抛 → 重试耗尽后 fallback 到备用模型。每次 retry/fallback 写 trace span。

flowchart TD A["invoke primary
调用主模型"] --> B{"成功?
是否抛 ModelError"} B -->|成功| C["返回 ModelResponse
正常返回响应"] B -->|失败| D{"error_type 可重试?
TIMEOUT/RATE_LIMIT/PROVIDER_UNAVAILABLE"} D -->|否 AUTH_ERROR/BAD_REQUEST| E["立即抛出 ModelError
不重试，不记 retry span"] D -->|是| F["record_retry span
记录重试 span（model 的子 span）"] F --> G["exponential backoff
指数退避等待"] G --> H{"attempt ≤ max_retries?
还有重试次数"} H -->|是| A H -->|否 耗尽| I{"有 fallback?
是否配置备用模型"} I -->|无| J["抛出最后 ModelError
无备用模型，失败抛出"] I -->|有| K["record_fallback span
记录 fallback span"] K --> L["invoke fallback
调用备用模型（仅一次）"] L --> M{"成功?
备用模型是否成功"} M -->|成功| C M -->|失败| J

③ ToolRuntime 权限与风险控制

执行前依次做 schema 校验 → permission check → risk policy check；所有路径（成功/失败/拒绝/确认/阻断）都写 audit log 和 tool span。

flowchart TD A["execute name args ctx
开始执行工具"] --> B["start_span TOOL
开启工具 span（agent 的子 span）"] B --> C["registry.get
查找工具"] C --> D{"工具存在?
是否已注册"} D -->|否| E["ToolResult ERROR
NOT_FOUND 工具不存在"] D -->|是| F["schema validation
按 input_schema 校验参数"] F --> G{"校验通过?
参数是否合法"} G -->|否| H["ToolResult ERROR
VALIDATION_ERROR 参数错误"] G -->|是| I["ToolPolicy.check
权限与风险判断"] I --> J["record_policy span
记录策略 span（tool 的子 span）"] J --> K{"decision?
策略决策结果"} K -->|read_only / write 有 scope| L["ALLOWED
允许执行"] K -->|write 无 scope| M["ToolResult DENIED
PERMISSION_DENIED 权限不足"] K -->|external_side_effect 需确认| N["ToolResult REQUIRES_CONFIRMATION

高风险阻断，不执行"] L --> P["handler 执行

④ ToolPolicy 风险决策树

按 risk_level 决定：read_only 放行；write 查 scope；external_side_effect 查 scope 且需确认；destructive 默认阻断。

⑤ trace tree 结构（一次含 retry 与工具调用的任务）

AgentRunner 是 root，model/tool 是 agent 子 span，retry/fallback 是 model 子 span，policy/audit 是 tool 子 span。
